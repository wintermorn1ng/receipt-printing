# 概述
这是一个纯本地的 "点单辅助系统"， 它的作用是帮助小餐饮业的店主简单的管理用户的点单，以及提供快捷的点单系统，点单后支持在蓝牙打印机上打印小票，作为叫号系统的依据。

# 设计原则
- 该系统的用户是普通操作人员，整个系统应该尽量简洁和容易理解，不需要繁琐的配置。
- 点单界面必须直观而高效，使用户可以在快节奏的点单中从容的使用。
- 系统必须足够健壮，对各种异常情况进行处理，保证主要功能的可靠运行

# 目标环境
安卓手机与安卓平板

# 限定技术栈
- flutter

# 功能模块

## 1. 菜单功能
- 拉平结构，无分类
- 核心字段：名称（必填）、价格（可选）、图片（可选）
- 网格展示，支持排序调整
- SQLite 本地持久化

详见：[[菜单功能 spec]]

## 2. 点单功能
- **点击即下单**，无需确认
- 大按钮网格布局，便于快速点击
- 取餐号每天自动从 1 开始，支持手动重置和设置起始号
- 一次下单 = 一个菜品，打印一张票

详见：[[点单功能 spec]]

## 3. 小票打印功能
- 点单后立即打印
- 小票内容：取餐号（必打）、菜品名（必打）、时间（可选）、店名（可选）
- 支持单打印机打两联（顾客联+厨房联）
- 预留双打印机扩展接口
- 蓝牙自动重连、失败告警

详见：[[小票打印功能 spec]]

## 4. 日总结功能
- 支持切换日期查看历史
- 核心统计：订单总数、各菜品销量
- 可选展示：时间段分布
- 数据长久保留

详见：[[日总结功能 spec]]
