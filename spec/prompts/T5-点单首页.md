# T5: 点单首页（含取餐号）

## 任务目标
实现核心点单界面和取餐号管理。**这是整个应用的核心功能**。

## 设计参考
- `/Users/victor/Documents/codes/flutter/receipt-printing/spec/点单功能 spec.md`

## 核心流程
```
用户点击菜品 → 获取当前取餐号 → 创建订单 → 打印小票 → 取餐号+1 → Toast反馈
```

## 详细步骤

### 1. TicketService (`lib/services/ticket_service.dart`)
管理取餐号：
```dart
class TicketService {
  // 获取当前取餐号
  Future<int> getCurrentTicketNumber();

  // 递增取餐号（下单成功后调用）
  Future<int> incrementTicketNumber();

  // 重置取餐号为1
  Future<void> resetTicketNumber();

  // 设置起始号
  Future<void> setTicketNumber(int number);

  // 检查是否需要每日重置
  Future<void> checkDailyReset();

  // 获取上次重置日期
  Future<String?> getLastResetDate();
}
```
存储使用 settings 表：
- `current_ticket_number`: 当前取餐号
- `last_reset_date`: 上次重置日期（YYYY-MM-DD）

### 5. HomeScreen (`lib/screens/home_screen.dart`)
界面布局：
```
┌─────────────────────────────────────┐
│  [#128] 当前取餐号        [重置▼]   │  ← TicketNumberDisplay
├─────────────────────────────────────┤
│  ┌─────────┐  ┌─────────┐           │  ← 菜品网格
│  │  牛肉面  │  │  炸酱面  │           │
│  │  ¥15    │  │  ¥12    │           │
│  └─────────┘  └─────────┘           │
│           ...                       │
├─────────────────────────────────────┤
│  [菜单管理]  [打印设置]  [日总结]    │  ← BottomNavigationBar
└─────────────────────────────────────┘
```

交互：
- 点击菜品：立即下单，显示 loading，成功后 Toast
- 取餐号区域：弹出菜单（重置、设置起始号）
- 重置：二次确认对话框
- 设置起始号：数字输入对话框
- 底部导航：切换页面

### 6. 启动检查
在 `main.dart` 或首页 `initState` 中：
```dart
// 检查是否需要每日重置取餐号
await ticketService.checkDailyReset();
```

## 技术要求
- 下单过程要防重复点击（loading 状态）
- Toast 使用 fluttertoast 或 ScaffoldMessenger
- 网格适配不同屏幕尺寸（手机 2列，平板 3-4列）
- 大按钮：最小高度 100dp，易于点击
