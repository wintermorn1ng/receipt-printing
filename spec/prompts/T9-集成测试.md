# T9: 集成测试与 Bug 修复

## 任务目标
确保各模块协同工作，修复发现的问题。

## 测试场景

### 场景 1: 首次安装完整流程
1. 全新安装应用
2. 进入首页，检查取餐号是否为 1
3. 进入菜单管理，添加 3-5 个菜品
4. 返回首页，检查菜品显示
5. 点击菜品下单
6. 检查：
   - 小票是否正确打印（内容、联数）
   - 取餐号是否 +1
   - Toast 是否显示
7. 连续点击多个菜品，检查订单是否正确
8. 进入日总结，检查数据是否正确

### 场景 2: 取餐号重置
1. 当前取餐号为 5
2. 点击重置，确认二次确认弹窗
3. 检查取餐号是否变为 1
4. 下单，检查小票是否为 #1

### 场景 3: 每日自动重置
1. 修改系统日期为明天
2. 重启应用
3. 检查取餐号是否自动重置为 1

### 场景 4: 打印机断连处理
1. 正常连接打印机
2. 关闭打印机蓝牙
3. 尝试下单
4. 检查错误提示是否友好
5. 重新开启打印机蓝牙
6. 检查是否能自动/手动重连

### 场景 5: 日总结数据准确性
1. 一天内下单 10 次（多个不同菜品）
2. 进入日总结，核对：
   - 订单总数是否为 10
   - 各菜品数量是否正确
   - 时间段分布是否合理
3. 切换到昨天（应该无数据）
4. 切换回今天，数据是否正常

### 场景 6: 边界情况
1. 无菜品时首页显示（空状态）
2. 菜品名称为空时保存（表单验证）
3. 删除正在被引用的菜品（应允许，订单已冗余存储 dishName）
4. 大量菜品时的网格滚动性能
5. 大量订单时日总结的加载性能

# E2E 测试配置（可选）

本项目支持使用 Patrol 进行端到端测试。Patrol 是 Flutter 生态中成熟的 E2E 测试框架，特别适合测试原生 UI 元素、硬件交互等场景。

## 安装步骤

### 1. 安装 patrol_cli（全局）
```bash
dart pub global activate patrol_cli
```

### 2. 添加依赖
在 `pubspec.yaml` 的 `dev_dependencies` 中添加：
```yaml
dev_dependencies:
  patrol: ^3.6.0
```

### 3. iOS 配置
确保 iOS 项目的 `ios/Runner.xcworkspace` 已启用 `nativeAutomation`：
- 打开 Xcode
- 在 Test 目标中勾选 "Allow UI Interaction"

### 4. 创建测试文件
在 `test/` 目录下创建 E2E 测试，例如 `test/e2e/app_test.dart`：
```dart
import 'package:flutter_test/flutter_test.dart';
import 'package:patrol/patrol.dart';

void main() {
  patrolTest('完整下单流程', ($) async {
    // 等待应用启动
    await $.pumpWidgetAndSettle();

    // 检查首页取餐号
    expect(find.text('1'), findsOneWidget);

    // 点击菜单管理
    await $.tap(find.byIcon(Icons.restaurant_menu));
    await $.pumpAndSettle();

    // 添加菜品
    await $.tap(find.byIcon(Icons.add));
    await $.pumpAndSettle();
    // ... 填写菜品表单并保存

    // 返回首页
    await $.tap(find.byIcon(Icons.arrow_back));
    await $.pumpAndSettle();

    // 点击菜品下单
    await $.tap(find.byType(DishGridItem).first);
    await $.pumpAndSettle();

    // 验证取餐号已更新
    expect(find.text('2'), findsOneWidget);
  });
}
```

## 运行测试

```bash
# 运行所有 E2E 测试
patrol test

# 运行指定测试
patrol test test/e2e/app_test.dart

# 指定设备
patrol test -d "iPhone 16 Pro"
```

## 适用场景

Patrol 特别适合测试以下场景：
- 完整用户流程（如首次安装 → 添加菜品 → 下单 → 打印）
- 蓝牙打印流程（需要真机）
- 系统权限弹窗交互（如蓝牙权限、通知权限）
- 多页面导航和数据传递

## 注意事项

1. **真机/模拟器**：蓝牙相关测试需要真机
2. **测试隔离**：每次测试后应清理状态，避免数据污染
3. **执行速度**：E2E 测试相对较慢，建议优先使用单元测试验证业务逻辑
