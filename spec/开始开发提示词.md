## 开始开发任务

现在开始基于已有的 Spec 文档进行 Flutter 点单辅助系统的开发。

### 前置条件

Spec 文档已全部完成，位于 `/Users/victor/Documents/codes/flutter/receipt-printing/spec/` 目录：
- `设计spec.md` - 项目概述和架构
- `菜单功能 spec.md` - 菜单功能详细设计
- `点单功能 spec.md` - 点单功能详细设计
- `小票打印功能 spec.md` - 打印功能详细设计
- `日总结功能 spec.md` - 日总结功能详细设计
- `开发计划.md` - 任务划分和开发计划
- `prompts/` - 各任务的详细 Coding Agent 提示词

### 你的任务

1. **创建开发团队**
   - 使用 `TeamCreate` 创建团队，命名为 `receipt-printing-dev`
   - 团队描述：Flutter 点单辅助系统开发团队

2. **组建协作团队**
   创建以下 teammates 并行工作：

   | 角色 | 类型 | 任务 |
   |------|------|------|
   | `architect` | Plan | 负责 T1-T3：项目初始化、数据库、模型定义 |
   | `menu-dev` | general-purpose | 负责 T4：菜单管理功能 |
   | `order-dev` | general-purpose | 负责 T5：点单首页（含取餐号）|
   | `print-dev` | general-purpose | 负责 T6：小票打印功能 |
   | `summary-dev` | general-purpose | 负责 T7：日总结功能 |

3. **分配任务**
   - 使用 `TaskCreate` 创建所有开发任务
   - 为每个任务设置依赖关系
   - 使用 `TaskUpdate` 将任务分配给对应的 teammate

4. **协调开发**
   - 定期查看 `TaskList` 了解进度
   - 解决任务间的依赖阻塞
   - 确保各模块接口一致

5. **执行顺序**
   ```
   阶段一（必须串行）：
   architect 完成 T1 → T2 → T3

   阶段二（可并行，但需等待阶段一完成）：
   menu-dev 执行 T4
   order-dev 执行 T5（依赖 T4 的菜单数据接口）
   print-dev 执行 T6（依赖 T5 的下单接口）

   阶段三：
   summary-dev 执行 T7
   architect 或任意 dev 执行 T8

   阶段四：
   全体成员协作 T9 集成测试
   architect 执行 T10 代码整理
   ```

6. **关键协作点**
   - 数据库表结构（T2 完成后需同步给所有人）
   - 菜品模型定义（T3 完成后 menu-dev 和 order-dev 需要）
   - 订单服务接口（T5 完成后 print-dev 和 summary-dev 需要）

### 注意事项

1. 每个 teammate 启动时，先读取对应的 prompt 文件（位于 `spec/prompts/`）
2. 阶段一完成后，再启动阶段二的 teammates
3. 定期广播进度，确保各模块接口对齐
4. 遇到问题立即暂停并汇报，不要连续尝试超过 3 次
5. 所有代码修改后，同步更新 spec 文档

### 开始执行

请立即开始：
1. 创建团队
2. 启动 `architect` 执行 T1-T3
3. 待基础架构完成后，再启动其他 teammates

## 使用示例

当你把上面的内容发给 Claude Code 后，它会执行：

1. `TeamCreate` 创建团队
2. `TaskCreate` 创建所有任务
3. `Agent` 启动 architect 开始 T1
4. 收到 architect 完成通知后，继续分配后续任务
5. 通过 `TaskList` 监控整体进度
6. 通过 `SendMessage` 协调各 teammate 的接口对接
