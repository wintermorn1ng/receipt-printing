# 点单功能

## 设计原则
- 极致效率：点击即下单，无需确认
- 界面直观：大按钮网格布局，易于快速点击
- 极简操作：无数量修改、无删除、无备注

## 取餐号规则

### 取餐号格式
- 纯数字，从 1 开始递增
- 每天 00:00 自动重置为 1
- 当前取餐号在界面上显著位置展示

### 取餐号管理
1. **自动重置**：每天首次启动应用时自动重置为 1
2. **手动重置**：提供重置按钮，点击后二次确认
3. **手动设置**：支持手动输入起始号（如从 100 开始）

### 重置确认弹窗
```
┌────────────────────────────┐
│        确认重置            │
│  当前取餐号为 #15          │
│  重置后将变为 #1           │
│                            │
│   [取消]      [确认重置]   │
└────────────────────────────┘
```

## 下单流程

1. 操作员点击菜品按钮
2. 系统立即：
   - 分配当前取餐号
   - 打印小票（顾客联 + 厨房联）
   - 取餐号 +1
   - 记录订单数据（用于日总结）
3. 界面显示短暂成功提示

## 订单数据结构

```dart
class Order {
  final int id;              // 订单ID
  final int ticketNumber;    // 取餐号
  final int dishId;          // 菜品ID
  final String dishName;     // 菜品名称
  final DateTime createdAt;  // 下单时间
}
```

**注意**：一次下单只包含一个菜品。即使客人点多个相同菜品，也是多次点击，生成多个订单，打印多张票。

## 交互说明

| 操作 | 效果 |
|------|------|
| 点击菜品按钮 | 立即下单，打印小票，取餐号+1 |
| 点击取餐号区域 | 弹出菜单：手动设置起始号 |
| 点击重置按钮 | 弹出二次确认弹窗 |
| 底部导航 | 切换至其他功能页面 |

## 成功反馈

下单成功后，在屏幕底部显示 Toast 提示：
- 显示内容：`#128 牛肉面 打印成功`
- 自动消失：1.5 秒
